Chat Bot for your Rails Application: Part 2

February 23, 2017February 24, 2017 Rakesh Avula artificial intelligence, rails, ruby
Share1

In the previous post we have developed an electronics catalog app as a basic setup with models as phones, televisions, and laptops.

In this 2nd post of 3 part series, we are going to develop chat UI, model associations and display of messages in the chat window.

This is how the app is going to look like by the end of this post.

Let’s get started.

First, we need to store the conversation between the user and the chatbot in the database. So we are going to create a model called “bot_action.rb” using the rails model generator.

Since the user will be posting many messages and we want the bot to perform an action based on the input given to it by the user, we are going to have a ‘has_many’ relation between the two models.

$rake db:migrate
 

In the generated model file we are going to add a presence validation and a length of input validation.

class BotAction < ApplicationRecord
  belongs_to :user
  validates_presence_of :user_input, message: "can't be empty"
  validates_length_of :user_input, minimum: 2, maximum: 200, message: "accepts only 2 - 200 characters"
end

We need the chat window to be displayed on every page so that user can input commands from any page that he visits. For this, we are going to create a new action called ‘new_bot_action’ in application controller.

path: app/controllers/application_controller.rb

class ApplicationController < ActionController::Base
  protect_from_forgery with: :exception
  before_action :new_bot_action
 
def new_bot_action
    if current_user.present?
      @bot_action = BotAction.new
      @bot_chat = current_user.bot_actions.last(2)
    end
  end
end

The before action callback is added to ensure that our new object of BotAction is built before any other actions. This way we can have the user input ready to go as soon as the application starts.

We are going to have two instance variables as @bot_action for our new object and @bot_chat to hold information of last recorded chat that we can access in application view.

Restart rails server and check. The chat window should now be displayed to the right in every page we visit.

Since we want the response to be displayed immediately without any page reload, we have used ‘remote: true’ with ‘form_for’ in the ‘_chat_bot.html.haml’ partial earlier. This ensures that each request we make to the ‘process_user_input’ action in ‘bot_actions_controller.rb’ is of js type.

class BotActionsController < ApplicationController
  def process_user_input
    @bot_action = current_user.bot_actions.build(bot_action_params)
    if @bot_action.save
      @bot_action.update_attribute(:bot_response, 'This is a place holder response')
      @return_user_input = @bot_action.user_input
      @return_bot_response = @bot_action.bot_response
    end
    respond_to do |format|
      format.js
    end
  end
 
private
 
def bot_action_params
    params.require(:bot_action).permit(:user_input, :user_id)
  end
end

Moving ahead, we are going to display the input by the user and the chatbot’s response in the chat window. For now, let’s have a placeholder response for the chatbot. We will work on the bot response after the chat display part is done.

Since, the post request to ‘process_user_input’ is done as js the response is also expected as js. So, we are going to rename ‘process_user_input.html.haml’ to ‘process_user_input.js.haml’ and replace its content with the following code.

Here, we are checking that our @bot_action is saved properly and that no validation errors occurred. If it’s saved properly we are appending ‘user_input’ partial and ‘bot_response’ partials to ‘.chat’ class in ‘_chat_bot.html.haml’ using javascript.

We are passing ‘@return_user_input’ variable as ‘return_user_input’ and ‘@return_bot_response’ as ‘return_bot_response’ into the partials respectively.

The animate method on ‘.panel-body’ class is used to scroll down the chat’s panel body, as the partials keep appending.
